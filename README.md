# apps-aus10-pages

本仓库用于归档和发布多个独立的 HTML 页面。内容以静态网页形式存放，不涉及特定域名或网站。

## 项目简介

这是一个通用型 HTML 页面归档仓库，主要用途包括：

- 存放各类独立 HTML 文件
- 提供静态页面的版本管理
- 便于通过 GitHub Pages 或其他方式直接访问

仓库本身不针对任何具体项目或用途，仅作为页面资源的集中存储与分发点。

## 目录说明

```
.
├── index.html          # 仓库首页（如有）
├── pages/              # 页面存放目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

各目录用途如下：

- `pages/`：存放主要 HTML 页面文件
- `assets/`：存放页面引用的静态资源文件
- 根目录：可放置首页或入口文件

## 页面归档说明

- 每个页面均为独立 HTML 文件，可单独访问
- 页面间无强制依赖关系
- 资源文件建议使用相对路径引用
- 如需新增页面，直接在 `pages/` 目录下添加即可

## 维护说明

- 添加新页面时，注意保持目录结构清晰
- 如有公共资源，建议统一放置在 `assets/` 目录下
- 页面命名建议使用英文小写，以短横线分隔
- 如需删除或更新页面，直接操作对应文件即可

## 其他

本仓库仅用于页面归档与发布，不涉及任何商业用途或具体业务逻辑。如有问题，欢迎提交 Issue 或 Pull Request。
